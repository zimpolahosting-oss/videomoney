# Video&Money

Flutter Android app with Firebase Authentication, Firestore storage, Material Design 3 UI, and placeholder rewarded ads.

## Included features

- Email/password sign in and sign up with Firebase Authentication
- Firestore `users` collection with:
  - `uid`
  - `email`
  - `coins`
  - `videosWatched`
  - `createdAt`
- Firestore `payouts` collection with:
  - `userId`
  - `coinsRequested`
  - `status`
  - `createdAt`
- Bottom navigation:
  - `Home`
  - `Earn`
  - `Wallet`
  - `Profile`
- Rewarded ad integration using the native Appodeal Android SDK
- Reward logic: user receives `200` coins after a successful rewarded ad
- Payout request flow stored in Firestore

## Setup

1. Install Flutter and Android Studio.
2. Create a Firebase project.
3. Add an Android app with package name `com.videomoney.app`.
4. Use the included `android/app/google-services.json` for Firebase Android configuration.
5. Keep your AdMob App ID configured in `android/app/src/main/AndroidManifest.xml`.
6. Set your Appodeal app key in `android/local.properties` using `appodeal.appKey=YOUR_APPODEAL_APP_KEY`, or export `APPODEAL_APP_KEY` before building.
7. Enable:
   - Firebase Authentication with `Email/Password`
   - Cloud Firestore
8. Optional: copy `android/local.properties.example` to `android/local.properties` and set your local SDK paths if they are not already generated by Flutter.
9. Run:

```bash
flutter create .
flutter pub get
flutter run
```

## Firestore rules example

Use secure rules that limit access to the signed-in user. Example starter rules:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    match /payouts/{payoutId} {
      allow create: if request.auth != null
        && request.resource.data.userId == request.auth.uid;
      allow read: if request.auth != null
        && resource.data.userId == request.auth.uid;
      allow update, delete: if false;
    }
  }
}
```

## Notes

- The Android project expects an Appodeal app key in `android/local.properties` or the `APPODEAL_APP_KEY` environment variable.
- Payout requests subtract the requested coin amount from the user balance immediately and store the request with status `pending`.
- Run `flutter create .` once on your machine to regenerate any missing native wrapper files before the first build.
- Firestore may ask you to create a composite index for the payout history query (`userId` + `createdAt`).
